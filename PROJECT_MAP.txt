خريطة المشروع - ELAWADI GROUP
آخر تحديث: 2026-05-01

1) وصف سريع
- المشروع عبارة عن موقع شركة تصميم داخلي/معماري.
- المحتوى الأساسي ديناميكي ويأتي غالبًا من الملف: data/content.json
- يوجد JavaScript مشترك مسؤول عن الترجمة والتحميل الديناميكي: js/main.js
- توجد صفحات HTML مستقلة لكل قسم من الموقع.

2) الملفات الأساسية
- index.html
  الصفحة الرئيسية.
  تحتوي على الهيرو، الخدمات، الإحصائيات، CTA، والفوتر.
  تم إزالة قسم المشاريع المكرر منها، وبقي عرض واحد فقط لبطاقات المشاريع داخل قسم الخدمات.

- courses.html
  صفحة الكورسات.
  كانت تحتوي نصوص مكسورة بعلامات استفهام وتم إصلاح النصوص الظاهرة بها.
  تعرض الكورسات من content.json.

- portfolio.html
  صفحة عرض كل المشاريع.
  تعتمد على portfolio داخل data/content.json.

- featured-projects.html
  صفحة مشاريع مميزة منفصلة.
  تعرض أول مشاريع من portfolio.

- project-detail.html
  صفحة تفاصيل المشروع عند الضغط على أي مشروع.

- services.html
  صفحة الخدمات.

- about.html
  صفحة عن الشركة.

- blog.html
  صفحة المدونة.

- contact.html
  صفحة التواصل.

- checkout.html
  صفحة شراء/الاشتراك في الكورس.

- admin.html
  لوحة الإدارة لتعديل المحتوى.

3) ملفات البيانات
- data/content.json
  أهم ملف بيانات في المشروع.
  يحتوي على:
  branding
  hero
  about
  services
  courses
  contact
  social
  portfolio
  portfolioSections
  ui

- data/messages.json
  غالبًا لتخزين رسائل التواصل.

- data/sessions.json
  غالبًا لجلسات/بيانات إدارية أو تشغيلية.

4) JavaScript المهم
- js/main.js
  الملف المشترك الأهم في الموقع.
  مسؤول عن:
  تحميل المحتوى
  الترجمة
  ربط بيانات التواصل
  بناء الجاليري/المشاريع
  سلايدر الهيرو
  مزامنة المحتوى

  دوال مهمة داخله:
  - loadDynamicContent()
  - applyLanguageContent()
  - updateDynamicGrids()
  - renderHomeHeroSlides()
  - getMediaUrl()
  - syncContactBindings()

5) Python / Backend
- flask_app.py
  تطبيق Flask أساسي.

- app_server.py
  سيرفر/تشغيل إضافي أو بديل حسب بيئة النشر.

- wsgi.py
  نقطة تشغيل WSGI.

- passenger_wsgi.py
  خاص ببيئات استضافة تستخدم Passenger.

- simple_server.py
  سيرفر بسيط للتجربة.

6) الأصول
- uploads/
  كل الصور والفيديوهات والملفات المرفوعة.

- css/style.css
  الستايل الأساسي المشترك.

7) ملاحظات مهمة جدًا
- main.js عليه version query في الصفحات مثل:
  js/main.js?v=20260501-1
  والسبب هو كسر الكاش بعد أي تعديل مهم.

- ملف .htaccess يفعّل كاش قوي للملفات الثابتة، لذلك عند تعديل JS/CSS يجب غالبًا تغيير رقم النسخة.

- الصفحة الرئيسية كانت لا تعرض المشاريع بسبب خطأ JavaScript في main.js وتم إصلاحه.

- صفحة الكورسات كان بها نصوص ثابتة مكسورة داخل نفس ملف HTML وتم إصلاحها.

8) مسار فهم سريع للمشروع
- ابدأ من index.html لفهم الواجهة الرئيسية.
- ثم افتح js/main.js لفهم من أين يأتي المحتوى وكيف يتم حقنه داخل الصفحات.
- ثم افتح data/content.json لفهم النصوص والبيانات المستخدمة فعليًا.
- بعد ذلك راجع portfolio.html و project-detail.html لمعرفة دورة عرض المشاريع.
- ثم راجع courses.html و checkout.html لمعرفة دورة الكورسات.

9) لو هنعدل مستقبلاً
- أي تعديل محتوى: راجع data/content.json أولًا.
- أي تعديل سلوك عام أو ترجمة أو ربط بيانات: راجع js/main.js.
- أي تعديل شكل صفحة بعينها: راجع ملف HTML الخاص بها ثم css/style.css إذا لزم.
- بعد أي تعديل مهم:
  1. راجع التشخيصات.
  2. اختبر الصفحة المتأثرة.
  3. حدّث version الخاص بـ main.js أو css لو الكاش ممكن يسبب مشكلة.

10) آخر تعديلات موثقة
- إصلاح سبب اختفاء المشاريع من الصفحة الرئيسية.
- إزالة قسم المشاريع المكرر من الصفحة الرئيسية.
- إصلاح النصوص المكسورة في صفحة الكورسات.
